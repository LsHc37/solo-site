# Retro Gigz Web

Retro Gigz public site + admin panel built with Next.js App Router, NextAuth credentials auth, and SQLite.

## Stack

- Next.js 16 (App Router)
- React 19
- TypeScript
- Tailwind CSS 4
- NextAuth v5 (credentials provider)
- better-sqlite3
- Vitest

## Quick Start

1. Install dependencies

```bash
npm install
```

2. Run in development

```bash
npm run dev
```

3. Open

http://localhost:3000

## Scripts

```bash
npm run dev          # Start dev server
npm run build        # Build production bundle
npm run start        # Start production server
npm run lint         # Run ESLint
npm run test         # Run Vitest in watch mode
npm run test:run     # Run Vitest once
npm run admin:bootstrap -- --email you@example.com --password your-password
```

## Authentication and Roles

- Login uses credentials auth (`email` + `password`).
- If an email does not exist, account creation is supported via first-time sign-in flow, but now requires explicit user consent.
- Admin-only areas are under `/admin` and are protected by role checks plus an extra admin portal code challenge.

### Admin Login

- Admins sign in at `/login?mode=admin&callbackUrl=/admin`.
- Admin access requires both the account password and the admin portal code.
- Default admin portal code: `1357`
- You can override the portal code with the `ADMIN_PORTAL_CODE` environment variable.

### Bootstrap First Admin

Promote an existing user or create a new admin account:

```bash
npm run admin:bootstrap -- --email you@example.com --password your-password
```

Behavior:

- Existing user: role is updated to `admin`
- New user: admin user is created (password required)

## Security and Ops Notes

- Credentials auth includes IP/email throttling, lockouts, and auth event logging.
- Admin code editor is disabled in production.
- File uploads are validated with extension/type/size limits.
- File deletion uses safer flows (trash by default, explicit permanent delete option).

## Admin-Managed Public Content

Public pages consume runtime-managed content/settings from:

- `site_settings`
- `content_blocks`

This powers maintenance mode, announcement banner, and editable hero/privacy text blocks.

## SEO

- `robots.txt` generated by `app/robots.ts`.
- `sitemap.xml` generated by `app/sitemap.ts`.
